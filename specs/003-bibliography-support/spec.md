# Feature Specification: Bibliography Support

**Feature Branch**: `[003-bibliography-support]`
**Created**: 2026-04-19
**Status**: Draft
**Input**: User description: "Bibliography — управление библиографией (bibtex-style)"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Inline Citations in Text (Priority: P1)

Академический автор пишет диссертацию и добавляет ссылки на источники в тексте с помощью команды `@cite`. Система автоматически нумерует источники и создаёт библиографическую запись в конце документа.

**Why this priority**: Без базовых inline-цитат вся функция библиографии не имеет смысла. Это ядро функции.

**Independent Test**: Можно полностью протестировать, создав Typst документ с несколькими цитатами и сгенерировав DOCX, в котором:
- Каждая цитата в тексте отображается как номер [1], [2] и т.д.
- В конце документа есть раздел "Список литературы" с правильно пронумерованными записями

**Acceptance Scenarios**:

1. **Given** Документ с цитатой `@[author2023]` в тексте, **When** Конвертер обрабатывает документ, **Then** В DOCX цитата отображается как [1] а в конце документа появляется запись author2023 под номером 1

2. **Given** Документ с несколькими цитатами `@[smith2020]` и `@[jones2021]` в тексте, **When** Конвертер обрабатывает документ, **Then** Цитаты нумеруются в порядке появления, и библиография содержит обе записи в соответствующем порядке

3. **Given** Документ с повторной цитатой `@[smith2020]`, **When** Конвертер обрабатывает документ, **Then** Повторная цитата использует тот же номер [1] что и первая

---

### User Story 2 - Bibliography File Parsing (Priority: P1)

Автор предоставляет BibTeX файл с библиографическими записями. Система корректно парсит все типы записей (article, book, inproceedings и т.д.) и извлекает необходимые поля.

**Why this priority**: Без парсинга BibTeX файла невозможно получить данные для генерации библиографии.

**Independent Test**: Можно протестировать, загрузив BibTeX файл с различными типами записей и проверив корректность извлечённых данных (author, title, year, journal и т.д.)

**Acceptance Scenarios**:

1. **Given** BibTeX файл с записью `@article{key2023, author={John Smith}, title={Article Title}, year={2023}, journal={Journal Name}}`, **When** Система парсит файл, **Then** Извлекаются все поля: author, title, year, journal

2. **Given** BibTeX файл с разными типами записей (article, book, inproceedings), **When** Система парсит файл, **Then** Каждый тип распознаётся и поля извлекаются корректно

3. **Given** BibTeX файл с кириллическими символами в author и title, **When** Система парсит файл, **Then** Кириллица сохраняется корректно в извлечённых данных

---

### User Story 3 - Formatted Bibliography Output (Priority: P2)

Сгенерированный DOCX содержит библиографию, оформленную по ГОСТ 7.32-2017. Каждая запись имеет правильный формат с нужными отступами, нумерацией и разделителями.

**Why this priority**: Пользователи ожидают, что библиография будет соответствовать стандартам оформления. Без этого функция непригодна для реальных академических документов.

**Independent Test**: Можно протестировать, сгенерировав DOCX и проверив визуально, что библиография отформатирована согласно ГОСТ (номер, отступ, автор, название, год, источник)

**Acceptance Scenarios**:

1. **Given** Документ с 5 цитатами, **When** DOCX сгенерирован, **Then** Библиография содержит все 5 записей в нумерованном списке с правильным форматированием

2. **Given** Документ с цитатами на разные типы источников (статья, книга, материалы конференции), **When** DOCX сгенерирован, **Then** Каждый тип записи отображается с соответствующим форматом (для книги: автор. Название. — Город: Издательство, Год. — Страницы)

3. **Given** Документ с длинными записями, **When** DOCX сгенерирован, **Then** Библиографические записи корректно переносятся на новую строку с стандартным красным отступом

---

### User Story 4 - Citation Style Options (Priority: P3)

Автор может выбрать стиль цитирования: числовой [1], автор-год (Smith, 2023), или библиографическая запись автора. Система применяет выбранный стиль ко всем цитатам в документе.

**Why this priority**: Разные академические традиции и журналы требуют разных стилей цитирования. Это расширяет применимость инструмента.

**Independent Test**: Можно протестировать, изменив стиль цитирования и проверив, что все цитаты в тексте и библиографии отображаются в соответствии с выбранным стилем.

**Acceptance Scenarios**:

1. **Given** Документ с цитатами и стилем "numeric", **When** DOCX сгенерирован, **Then** Цитаты в тексте отображаются как [1], [2], а библиография содержит записи с номерами

2. **Given** Документ с цитатами и стилем "author-year", **When** DOCX сгенерирован, **Then** Цитаты в тексте отображаются как (Smith, 2023), и библиография содержит записи отсортированные по авторам

---

### Edge Cases

- Что происходит, когда BibTeX запись не содержит обязательных полей (например, отсутствует author)?
- Как система обрабатывает дублирующиеся ключи в BibTeX файле?
- Что происходит, когда цитируется ключ, которого нет в библиографическом файле?
- Как обрабатываются специальные символы в BibTeX (LaTeX команды, например `\textit{}`)?
- Что происходит с очень длинными названиями статей в библиографии?

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: Система ДОЛЖНА поддерживать загрузку BibTeX файлов (`.bib` расширение) для определения библиографических записей
- **FR-002**: Система ДОЛЖНА распознавать Typst команду `#bibliography("file.bib")` для подключения библиографии
- **FR-003**: Система ДОЛЖНА поддерживать Typst команду `@[key]` для вставки inline-цитат
- **FR-004**: Система ДОЛЖНА автоматически нумеровать цитаты в порядке их появления в документе
- **FR-005**: Система ДОЛЖНА генерировать раздел "Список литературы" в конце DOCX документа
- **FR-006**: Система ДОЛЖНА форматировать библиографические записи согласно ГОСТ 7.32-2017
- **FR-007**: Система ДОЛЖНА корректно обрабатывать повторные цитаты (использовать существующий номер вместо создания новой записи)
- **FR-008**: Система ДОЛЖНА поддерживать основные типы BibTeX записей: article, book, inproceedings, techreport, misc
- **FR-009**: Система ДОЛЖНА извлекать стандартные поля BibTeX: author, title, year, journal, booktitle, publisher, pages, url, doi
- **FR-010**: При отсутствии библиографического файла система ДОЛЖНА выдавать предупреждение но продолжать конвертацию
- **FR-011**: При отсутствии записи для ключа цитаты система ДОЛЖНА выдавать предупреждение и использовать placeholder
- **FR-012**: Система ДОЛЖНА поддерживать кириллические символы в BibTeX файлах
- **FR-013**: Система ДОЛЖНА поддерживать выбор стиля цитирования через CLI флаг или конфигурацию

### Key Entities

- **BibliographyFile**: Представляет загруженный .bib файл с коллекцией записей
- **BibliographyEntry**: Отдельная библиографическая запись с типом (article, book и т.д.) и набором полей (author, title, year, etc.)
- **Citation**: Inline-маркер в тексте документа, ссылающийся на ключ записи
- **CitationStyle**: Параметры форматирования цитат (numeric, author-year)
- **BibliographySection**: Раздел в DOCX с отформатированным списком литературы

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Пользователь МОЖЕТ создать Typst документ с цитатами `@[key]` и bibliography file, конвертировать в DOCX и получить пронумерованные inline-цитаты [1], [2], [3]

- **SC-002**: Сгенерированный DOCX содержит секцию "Список литературы" с количеством записей равным количеству уникальных цитируемых ключей

- **SC-003**: Все BibTeX записи корректно парсятся и извлекают author, title, year для каждого типа записи (article, book, inproceedings)

- **SC-004**: Повторные цитаты одного и того же ключа отображаются с тем же номером, что и первая цитата

- **SC-005**: Библиографические записи в DOCX соответствуют формату ГОСТ 7.32-2017 (номер, отступ 1.25см, автор, название, год, источник)

- **SC-006**: Конвертация документа с bibliography завершается успешно за время < 5 секунд для документа с 50 цитатами

## Assumptions

- Пользователи уже имеют BibTeX файлы с библиографией в стандартном формате
- Основной use case — академические документы (диссертации, статьи) требующие ГОСТ форматирования
- Кириллическая библиография поддерживается (многие российские авторы используют кириллицу в BibTeX)
- Пользователи знакомы с базовыми командами Typst для цитирования (@[key])
- Максимальное количество записей в bibliography — несколько сотен (типичный максимум для диссертации)
- Библиография находится в отдельном файле, не inline в основном документе
