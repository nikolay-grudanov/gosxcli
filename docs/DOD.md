## Обязательные требования к конвертеру

### 1. Поддержка импорта пакетов
- Обработка `#import "@preview/modern-g7-32:0.2.0": ...`
- Поддержка внешних пакетов Typst

### 2. Обработка `#show` директив
- `#show: gost.with(...)` - параметры документа
- `#show table.cell.where(y: 0): ...` - условное форматирование
- `#set text(hyphenate: true)` - настройки текста

### 3. Содержимое документа

#### Заголовки с метками
```typst
= Литературный обзор <ch01>
== LLM-агенты в моделировании городской мобильности <ch01-llm-agents>
```
- Поддержка заголовков уровней 1-3
- Поддержка меток `<label>`
- Сохранение меток в IR

#### Ссылки на литературу
```typst
В работе @gao2023social традиционные ABM-системы...
Система @zhang2024rest "LLMob" представляет...
```
- Извлечение всех `@label` ссылок
- Разрешение ссылок

#### Сложные таблицы
```typst
#figure(
  table(
    columns: (17%, 17%, auto, auto, auto, auto, auto),
    stroke: 0.7pt,
    fill: (col, row) => if row == 0 { luma(220) } else { none },
    align: (col, row) => if row == 0 { center } else { left },
    table.header([Система], [Тип], ...),
    [MATSim], [Traditional ABM], [...],
    ...
  ),
  caption: [Сравнение систем...],
) <tbl:abm-comparison>
```
- Поддержка `columns` с разными ширинами
- Поддержка `stroke`
- Поддержка `fill` (условное заполнение)
- Поддержка `align` (условное выравнивание)
- Поддержка `table.header`
- Поддержка `caption` с меткой

#### Структура текста
```typst
= Введение

Моделирование городской мобильности представляет собой одну из ключевых задач...

*Цель работы.*

Целью настоящей работы является...

*Задачи работы.*

+ Проведение анализа существующих подходов...
+ Разработка архитектуры системы...
+ Реализация LLM-планировщика...
```
- Обычные абзацы
- Выделенные блоки `*Заголовок*`
- Маркированные списки с `+`

### 4. Структура файлов
- Обработка `#include "chapters/00-vvedenie.typ"`
- Рекурсивная загрузка включённых файлов
- Обработка относительных путей

### 5. Библиография
```typst
#bibliography("refs.bib")
```
- Обработка библиографии (минимально)

## Известные ограничения MVP

**ОК для MVP**:
- Не идеальное воспроизведение форматирования
- Прокси для сложных Typst конструкций
- Ручная доводка в Word

**НЕ ОК для MVP**:
- Потеря контента (текст должен быть)
- Поломанная структура
- Нечитаемый DOCX
- Краш на реальном файле
