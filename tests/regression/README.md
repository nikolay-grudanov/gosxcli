# Regression Tests

Этот каталог содержит регрессионные тесты для проверки стабильности конвертера Typst → DOCX.

## Назначение

Регрессионные тесты гарантируют, что изменения в коде конвертера не ломают существующую функциональность. Они сравнивают сгенерированные DOCX файлы с эталонными (golden) файлами.

## Структура

- `test_regression.py` — основные регрессионные тесты
- `conftest.py` — общие pytest fixtures
- `fixtures/` — Typst исходные файлы и golden DOCX файлы

## Golden Files

Golden (эталонные) файлы — это DOCX файлы, которые были сгенерированы конвертером и проверены вручную. Они используются как ожидаемый результат.

### Текущие golden файлы:

1. `fixtures/minimal/minimal_golden.typ` → `minimal_golden.docx`
   - Базовый документ с headings, paragraphs, lists, tables, figures, equations

2. `fixtures/complex_table/complex_table.typ` → `complex_table.docx`
   - Таблицы с colspan, rowspan, stroke, fill, align

3. `fixtures/equations/math-formulas.typ` → `math-formulas.docx`
   - Математические формулы разной сложности

## Запуск тестов

### Обычный запуск (сравнение с golden)

```bash
pytest tests/regression/
```

### Обновление golden файлов

Если вы внесли изменения в конвертер, которые влияют на вывод DOCX, и эти изменения являются ожидаемыми, вам нужно обновить golden файлы:

```bash
pytest tests/regression/ --update-golden
```

**Внимание:** Перед обновлением golden файлов убедитесь, что изменения в выводе являются корректными!

### Запуск конкретного теста

```bash
pytest tests/regression/test_regression.py::TestRegression::test_minimal_regression
```

## Что проверяется

Регрессионные тесты проверяют:

1. **Структура документа** — количество headings, paragraphs, tables, figures
2. **Пустые параграфы** — отсутствие избыточных пустых параграфов
3. **XML содержимое** — базовая проверка структуры XML (опционально)

## Когда обновлять golden файлы

Golden файлы нужно обновлять когда:

1. Вы добавили новую фичу в конвертер
2. Вы исправили баг, который влиял на вывод DOCX
3. Вы оптимизировали генерацию DOCX (но сохранили результат)

Golden файлы НЕ нужно обновлять когда:

1. Вы изменили только внутреннюю реализацию (вывод не изменился)
2. Вы случайно сломали что-то (тогда тесты должны провалиться!)

## Добавление новых regression тестов

Для добавления нового regression теста:

1. Создайте Typst файл в `fixtures/<category>/<name>.typ`
2. Сгенерируйте DOCX: `python scripts/generate_golden.py` (добавьте путь к файлу)
3. Проверьте вручную, что DOCX файл корректен
4. Добавьте путь к файлу в `tests/regression/conftest.py` в словари `golden_docx_path` и `golden_typst_path`
5. Добавьте тестовый метод в `test_regression.py`

## Устранение проблем

### Тесты проваливаются после изменений в конвертере

Если regression тесты проваливаются после ваших изменений:

1. Проверьте, что изменения в выводе являются ожидаемыми
2. Посмотрите на diff отчет в выводе pytest
3. Если изменения корректны, запустите `pytest tests/regression/ --update-golden`
4. Если изменения некорректны, исправьте код конвертера

### Golden файлы отсутствуют

Если golden файлы отсутствуют:

```bash
python scripts/generate_golden.py
```

### Конвертер не работает

Если конвертер не генерирует DOCX файлы:

1. Проверьте, что Typst файл существует
2. Запустите конвертер вручную с `--debug` для получения подробной информации
3. Проверьте, что все зависимости установлены

## Автоматизация

Для удобства есть Makefile цель:

```bash
make regression
```

Это запускает regression тесты.
